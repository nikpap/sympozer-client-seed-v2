About the ESWC2017 app

This app is available on the Web and on the App & Play stores.

Its data structure follows the newly published ScholarlyData Ontology, formerly Semantic Web Conference Ontology, that describes persons, organizations, publications and events of the conference. The application allows browsing these data. It is also a client-side mashup that can enrich them using other data sources (Twitter, DBLP) or search engines (Google, DuckDuckgo).

You can use it offline to browse the conference dataset. You will need an Internet connection for viewing data from online sources, as well as to vote for your preferred poster and demo.

Periodically (but not too often), you can update the conference dataset in the settings panel.

How it works

In 2017, the app architecture has been completely refactored:

  • Its core engine is based on RDF data (instead of JSON), internally served using the rdflib.js library. Feel free to open the console and watch the app sending SPARQL requests to the store. The conference dataset has been issued using cLODg (conference Linked Open Data generator) V2.
  • It fully runs on the client side, and relies on the Angular 2 JavaScript framework, using Web components and other advances. The design layer uses Angular Material.

Issues

Do not hesitate to report issues on the following GitHub repositories:

  • problems with the data (e.g. incorrect/missing information)
  • problems with the app (e.g. strange behaviors, typos in the titles, usability issues, suggestions of improvement...)
